# Telegram-бот учета продаж сотрудников магазина одежды

Бот принимает ежедневный текстовый отчет от администратора, автоматически
разбирает его (сотрудники, касса, онлайн-продажи, бонусы), считает планы
и выполнение, и предоставляет статистику и экспорт в Excel.

## Стек

- Python 3.12
- aiogram 3
- SQLAlchemy 2.0 (async, SQLite по умолчанию, легко переключить на PostgreSQL)
- pandas + openpyxl (экспорт в Excel)
- python-dotenv

## Установка

1. Распакуйте проект и перейдите в его директорию:

   ```bash
   cd tg_sales_bot
   ```

2. Создайте и активируйте виртуальное окружение:

   ```bash
   python3.12 -m venv venv
   source venv/bin/activate        # Linux / macOS
   venv\Scripts\activate           # Windows
   ```

3. Установите зависимости:

   ```bash
   pip install -r requirements.txt
   ```

4. Создайте файл `.env` на основе примера:

   ```bash
   cp .env.example .env
   ```

5. Откройте `.env` и укажите:
   - `BOT_TOKEN` — токен вашего бота, полученный у [@BotFather](https://t.me/BotFather)
   - `ADMIN_IDS` — Telegram user_id администраторов через запятую (узнать свой id можно
     у бота [@userinfobot](https://t.me/userinfobot)). Если оставить пустым — доступ
     к админ-командам и `/import` будет у всех пользователей.
   - при необходимости — `DEFAULT_PLAN_CONSULTANT` / `DEFAULT_PLAN_ONLINE` (планы по
     умолчанию для новых сотрудников).

## Запуск

```bash
python -m bot.main
```

При первом запуске бот автоматически создаст файл базы данных SQLite
(`sales_bot.db`) и все необходимые таблицы.

## Переход на PostgreSQL

Замените в `.env`:

```
DATABASE_URL=postgresql+asyncpg://user:password@localhost:5432/sales_bot
```

и установите драйвер:

```bash
pip install asyncpg
```

Больше никаких изменений в коде не требуется — SQLAlchemy сам работает
с любой указанной СУБД.

## Формат текстового отчета

Бот понимает как классический формат (значения через тире), так и вариант
без тире / с сотрудниками смены каждым на отдельной строке. Пример:

```
Дата: 13.07

Количество чеков - 64

Безнал - 553.900
Наличными - 245.000

Общая Выручка
План / Факт - 798.900

Средний чек - 13.741

Конверсия - 51%

Онлайн продажи: 76.930

Вообщем: 875.830

Клиентов вообщем: 124

Товара продано: 96

Бонус:
Алина - 2000
Диас - 2000

Смена:
Арман, Алина, Жасулан

Касса:
Арман - 399.450
Алина - 399.450

Смена открыта: 11:00
```

Логика разбора:

1. Из блока **Смена** извлекается список всех сотрудников смены.
2. Из блока **Касса** извлекаются суммы консультантов.
3. Сотрудники из смены, отсутствующие в блоке **Касса**, считаются
   **онлайн-менеджерами**.
4. Сумма из строки **Онлайн продажи** делится поровну между всеми
   онлайн-менеджерами (функция `distribute_online_sales` в
   `bot/services/distribution.py` — вынесена отдельно, чтобы алгоритм
   распределения можно было легко изменить, например, сделать
   пропорциональным).
5. Если сотрудника нет в базе — он создается автоматически с ролью
   `online` (если это онлайн-менеджер) или `consultant` (если он есть
   в кассе), и планом по умолчанию из `.env`.
6. Повторный импорт отчета за уже существующую дату **заменяет** старые
   данные (счетчик смен корректно откатывается перед заменой).

## Команды бота

### Пользовательские

| Команда | Описание |
|---|---|
| `/start` | Приветствие |
| `/help` | Список команд |
| `/import` | Запустить импорт отчета (следующим сообщением пришлите текст отчета) |
| `/stats` | Общий рейтинг сотрудников за все время |
| `/employee <Имя>` | Полная статистика по сотруднику |
| `/day` | Статистика за сегодня |
| `/month` | Статистика за текущий месяц |
| `/year` | Статистика за текущий год |
| `/top [N]` | Топ-N сотрудников по выполнению плана (по умолчанию 5) |
| `/export` | Выгрузить статистику всех сотрудников в Excel |

### Администраторские (доступны только `ADMIN_IDS`)

| Команда | Описание |
|---|---|
| `/add_employee Имя consultant\|online план` | Добавить сотрудника (или без аргументов — пошаговый диалог) |
| `/remove_employee Имя` | Удалить сотрудника |
| `/set_role Имя consultant\|online` | Изменить роль сотрудника |
| `/set_plan consultant\|online\|Имя сумма` | Изменить дневной план — для всей роли или для конкретного сотрудника |

## Формулы расчета

- `план = количество_смен × дневной_план_сотрудника`
- `процент_выполнения = продажи / план × 100`
- `средняя_касса = продажи / количество_смен`

## Логирование

Логи пишутся в stdout через стандартный модуль `logging` (уровень INFO).
При необходимости настройте вывод в файл в `bot/main.py`.
